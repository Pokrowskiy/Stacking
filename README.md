# Stacking
Исходный код для фокус-стекинга изображений. В репозитории находится код для обработки изображений тремя разными способами (Через крассический попиксельный лаплсасиан, пирамидальный анализ и через U-Net)

# Быстрый старт
## 1) Подготовка окружения
1) Склонируйте репозиторий
2) Создайте и активируйте виртуальное окружение
```
python -m venv venv
venv\Scripts\activate
```
3) Установите зависимости
```
pip install -r requirements.txt
```

## 2) Запуск из коммандной строки

Управление утилитой осуществляется через аргументы командной строки. Ниже приведено подробное описание всех доступных параметров конфигурации:

| Аргумент | Тип данных | Значение по умолчанию | Описание |
| :--- | :---: | :---: | :--- |
| `--method` | `str` | `None` | **Метод фокус-стекинга.** Допустимые варианты: `basic`, `pyramidal` или `ML`. Если аргумент опущен, утилита поочередно выполнит обработку всеми доступными методами для сравнения. |
| `--images_dir` | `str` | `'samples/images'` | **Путь к папке с исходным стеком.** Директория, где лежат кадры с разной фокусировкой, которые необходимо объединить. |
| `--output_dir` | `str` | `'samples/output'` | **Путь к папке для сохранения результатов.** Сюда будут записаны выровненные/обрезанные кадры, а также финальные полнофокусные изображения для каждого метода. Папка создается автоматически, если её нет. |
| `--use_metrics` | `str` | `None` | **Активация расчета метрик.** Сюда передается путь к эталонному изображению (например, `samples/images/expected.jpg`). Если флаг указан, программа автоматически рассчитает и выведет в консоль значения PSNR, SSIM и LPIPS для каждого метода относительно этого эталона. |
| `--console_timer_output` | `bool` | `True` | **Логирование производительности.** Включает или выключает подробный вывод в консоль времени выполнения каждого этапа (загрузка, ECC-выравнивание, инференс, постпроцессинг) и пикового потребления оперативной памяти (ОЗУ/VRAM). |

### Примеры комманд запуска с использованием примеров, лежащих в папке `samples/`

Запуск на изображениях с микромхемами(7 кадров 2048х1536)

`python main.py --images_dir samples/images/micro --output_dir samples/output/micro --use_metrics samples/expected/micro.jpg`

Запуск на изображениях с кристаллом(5 кадров 1332х1152)

`python main.py --images_dir samples/images/cryst --output_dir samples/output/csryst --use_metrics samples/expected/cryst.png`

## Формат входных данных
- В качестве входных данных программа принимает путь к папке с с изображениями различной глубины фокусировки. Изображения должны был проименованы в формате `N.разрешение`, где `N` - номер кадра. Кадры должны быть пронумерованы от переднего плана к заднему. Нарушенный порядок нумерации файлов может вызвать снижение качества обработки.

## Формат выходных данных
- Результатом работы программы является объединённое изображение в папке, указанной для сохранения результатов. Выходной файл будет подписан в формате `result_{Название_метода}.png`. 
- В случае, если используется сравнение с образцом, туда же сохраняются файлы `cropped_sample.png`(Образец, обрезанный для соотвествия разрешению изображения после выравнивания) и `diff_image_{Название_метода}.png`(Изборажение-разница для визуальной оценки различий между изображениями). Также в консоль будут выведены метрики сравнения.
